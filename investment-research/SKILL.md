### 🔴 腾讯财经兜底验证（2026-06-15实战教训）

采集完成后，**必须用腾讯财经验证**（尤其A股/国内公司）：
- 国内A股：https://gu.qq.com/sz000568 或 https://gu.qq.com/sh601899
- 港股：https://gu.qq.com/hk00941
- 美股/海外公司也优先走腾讯财经快照确认

验证后标注 `✅腾讯财经一致` 或 `❌腾讯财经不符→以腾讯财经为准`。
此条为强制兜底规则，不接受省略。

- **A股股价唯一权威：腾讯财经 `http://qt.gtimg.cn/q=`** + 代码（sz300750/sh601138格式）。价格偏差>1%立即重采
- **美股股价：CNBC/Yahoo为主，gtimg美股PE不准（非GAAP PE）**
- **任意两源偏差>20%** → 排查，以最权威者为准（公司IR > SEC Filing > CNBC > 其他）
- **PE<1 或 >5000** → 价格错，立即修正。注意腾讯API的PE字段需÷100
- **韩国股票**：만원→원单位陷阱（₩2,150,000不是₩215,000）
- **用户截图 > API > 记忆**

### 腾讯财经API调用规范（🔴 A股强制）

```python
import urllib.request
# 代码格式：深圳sz+代码，上海sh+代码
url = f'http://qt.gtimg.cn/q=sz300750'  # 宁德时代
resp = urllib.request.urlopen(url).read().decode('gbk')
fields = resp.split('~')
price = fields[3]      # 现价
prev_close = fields[4] # 昨收
pe_ttm = float(fields[45]) / 100 if fields[45] else None  # PE需÷100
```

**每次数据采集后，A股必须跑一遍腾讯API验证价格。偏差>1% → 报告内价格为腾讯数据。**

## 阶段三：合规QA（🔴 强制，阶段四之前执行）

### 3.1 免责声明验证

每份MD末尾必须包含免责声明。使用grep验证：

```bash
grep -l "免责声明" *.md  # 必须每份都返回
```

### 3.2 违禁词扫描（🔴 最高优先级）

扫描所有MD，禁止出现：**目标价、买入(评级义)、卖出(评级义)、持有(评级义)、仓位建议、上涨空间、增持、减持、强烈推荐**

例外（以下不算违规）：
- 免责声明段落中的否定表述（"本文档不构成投资建议"）
- 框架术语（Buffett"卖出四准则"指框架机制，非评级）
- 日常口语（"每卖出3辆电动车"中的"卖出"=sold）

```bash
grep -n "目标价\|买入\|卖出\|仓位建议" *.md | grep -v "免责\|不构成\|非投资\|禁止\|不代表" 
# 空输出=合规通过
```

### 3.3 内部一致性验算（6项强制）

| # | 检查项 | 公式 | 偏差 |
|---|--------|------|:--:|
| 1 | EPS × 股数 ≈ 净利 | `EPS × shares` | <1% |
| 2 | 股价 ÷ EPS ≈ PE | `price / eps` | <0.5% |
| 3 | 净利 < 营业利润 | `op profit > net profit` | 必须 |
| 4 | 概率加权 = Σ(p×t) | `p1×t1 + p2×t2 + p3×t3` | 精确 |
| 5 | PEG = PE ÷ 增速% | `pe / growth_rate` | <1% |
| 6 | 市值 = 股价 × 总股本 | `price × shares` | <1% |

## 周期股/矿业股强制标注（🔴 v3.0新增）

生成以下行业报告时，必须在所有四份报告中标注 **「⚠️周期高点PE陷阱」**：
- 矿业（金/铜/锂/铁矿石/煤炭）
- 石油天然气
- 航运
- 钢铁/电解铝
- 养猪/农产品

标注逻辑：当前低PE可能反映商品价格处于周期高位——EPS建立在不可持续的利润率上，而非永久性低估。引用历史案例（如2018年半导体Q2：PE 4-5x看似极便宜，随后EPS暴跌60%）。

## 实战经验（2026-06-15 六家A股批量）

### 批量报告教训

- **⚠️ Subagent批量上限**：单次 `delegate_task` 最多 **3家公司**。5家一个subagent必超时（实测600s）。拆分3+2两批。
- **字数验证用Python不用grep**：`grep -oP` 在WSL下对中文计数不准，必须 `python3 -c "import re;..."`
- **首次产出偏薄是正常的**：subagent并行时首版MD可能低于字数目标。走「薄版→用户反馈→定向扩写」，不一稿求完美。
- **🆕 旧文件干扰**：subagent发现目录已有旧版报告时可能复用而非重写。首次薄版（如紫金9079字 vs 目标26000+）通常因为这个。解决方法：在prompt中明确「即使目录已有文件也全量重写」。
- **工业富联/风华高科偏薄案例**：旧版报告残留→subagent复用短内容。解决：确认目录为空或明确全量重写。
